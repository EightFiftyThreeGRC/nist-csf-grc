Built in the open. On purpose.

EightFiftyThree GRC makes deliberate engineering trade-offs. Here is what it is — and what it is not — so you can decide whether it fits before you trust it with a program.

▢

A client-side trust boundary

The whole app runs in your browser. There is no custom application server to breach — sign-in and data access are enforced by your identity provider and Supabase Row-Level Security, scoped to your account. The honest trade-off: a browser-based tool trusts the browser, so it is built to manage a governance program, not to hold secrets.

⚙

Zero dependencies, no build

Plain HTML, CSS, and vanilla JavaScript — no framework, no bundler, no npm supply chain. The source you read is the source that runs. It loads instantly and will still open in a browser a decade from now.

⇪

Your data, portable by default

One-click JSON export puts your entire program in a file you keep outside the app. Point the tool at your own Supabase project to self-host it end to end. You are never locked in.

⊘

What it deliberately is not

Not a 3PAO or a substitute for an assessor's judgment. Not a system-of-record you would run without your own backups. Not multi-tenant SaaS. It is a fast, free, defensible way to stand up and run an 800-53 program.
